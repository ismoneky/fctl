# 预约时间段功能说明

## 📅 功能概述

在预约表单中，用户需要选择：
1. **预约日期** - 具体哪一天
2. **预约时间段** - 上午或下午

这样可以更精准地安排游客的参观时间，避免过度集中。

## ⏰ 时间段划分

### 🌅 上午场
- **时间范围**: 08:00 - 12:00
- **适合人群**: 喜欢早起、避开人群的游客
- **优势**: 空气清新、光线柔和、适合拍照

### 🌇 下午场
- **时间范围**: 12:00 - 18:00
- **适合人群**: 喜欢睡懒觉、下午出行的游客
- **优势**: 温度适宜、游览时间充足

## 🎨 UI设计特点

### 卡片式选择
- 两个并排的时间段卡片
- 包含图标（🌅 上午 / 🌇 下午）
- 显示具体时间范围
- 选中状态有明显的视觉反馈

### 选中效果
- ✅ 背景色变为浅蓝色 (#e8f4ff)
- ✅ 边框变为蓝色 (#4A90E2)
- ✅ 文字颜色变为蓝色
- ✅ 右侧显示蓝色勾选图标 (✓)
- ✅ 添加阴影效果

### 未选中效果
- ⭕ 灰色背景 (#f8f8f8)
- ⭕ 透明边框
- ⭕ 黑色文字
- ⭕ 无勾选图标

## 💾 数据结构

```javascript
formData: {
    bookingDate: '2026-03-15',     // 预约日期
    timePeriod: 'morning',          // 预约时间段: morning | afternoon
    // ... 其他字段
}
```

## ✅ 表单验证

### 必填验证
```javascript
// 验证预约日期
if (!this.formData.bookingDate) {
    uni.showToast({
        title: '请选择预约日期',
        icon: 'none'
    });
    return false;
}

// 验证预约时间段
if (!this.formData.timePeriod) {
    uni.showToast({
        title: '请选择预约时间段',
        icon: 'none'
    });
    return false;
}
```

## 📱 用户操作流程

```
1. 选择预约日期
   ↓
2. 点击时间段卡片（上午/下午）
   ↓
3. 卡片变为选中状态
   ↓
4. 可以重新点击切换时间段
   ↓
5. 继续填写其他信息
   ↓
6. 提交表单
```

## 🔄 与预约记录的联动

预约记录页面会显示完整的时间信息：

```
预约时间: 2026-03-15 上午 (08:00-12:00)
```

格式：`日期 + 时间段 + 具体时间范围`

## 🎯 使用场景

### 景区管理
- **流量控制**: 分时段限流，避免拥挤
- **资源调配**: 根据时段安排工作人员
- **数据分析**: 统计不同时段的热度

### 游客体验
- **自由选择**: 根据个人习惯选择时段
- **避开高峰**: 选择人少的时段
- **合理安排**: 配合其他行程安排

## 💡 后续优化建议

### 1. 动态时段
根据不同景区设置不同的时间段：
```javascript
timePeriods: [
    { label: '早场', time: '06:00-09:00', icon: '🌄' },
    { label: '上午', time: '09:00-12:00', icon: '🌅' },
    { label: '下午', time: '12:00-15:00', icon: '☀️' },
    { label: '傍晚', time: '15:00-18:00', icon: '🌇' }
]
```

### 2. 剩余名额显示
显示每个时段的剩余名额：
```html
<view class="period-quota">
    剩余: 120/200
</view>
```

### 3. 时段推荐
根据历史数据推荐最佳时段：
```html
<view class="period-tag">推荐</view>
<view class="period-tag">人少</view>
```

### 4. 实时更新
实时显示各时段的预约情况：
```javascript
// 获取时段预约数据
getTimePeriodData() {
    uni.request({
        url: '/api/booking/time-period',
        success: (res) => {
            // 更新各时段的剩余名额
        }
    })
}
```

### 5. 禁用时段
已满或不可用的时段自动禁用：
```html
<view class="time-period-item disabled">
    <view class="period-tag">已满</view>
</view>
```

## ⚠️ 注意事项

1. **必选项**: 时间段为必选项，不能为空
2. **单选**: 只能选择一个时间段
3. **可切换**: 可以随时切换时间段
4. **联动验证**: 提交时会同时验证日期和时间段
5. **数据提交**: 提交时需要包含 `timePeriod` 字段
